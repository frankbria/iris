## Why this matters

`npm audit --omit=dev` reports **1 critical + 3 high** advisories in runtime
dependencies. `simple-git@3.28.0` has three RCE advisories (option-parsing
bypass), and it backs the visual-baseline git integration. `ws@8.18.3` has an
uninitialized-memory-disclosure and a memory-exhaustion DoS advisory, and `ws`
is the transport for the `iris connect` JSON-RPC server — the most
network-exposed surface in the project. All four are fixable with a single
`npm audit fix` (no major-version bumps required). Leaving them is the
lowest-effort, highest-value security win available.

## Steps

### Step 1: Establish the pre-change test baseline

Before changing anything, record the current suite result so you can prove the
upgrade introduced no regression. Run:

`npx jest 2>&1 | tail -5`

Expected (this is the known baseline — 2 pre-existing failures unrelated to
this plan): `Tests: 2 failed, 22 skipped, 532 passed, 556 total`. Write down
these numbers.

### Step 2: Apply the fix

Run `npm audit fix` (NOT `--force`).

**Verify**: `npm audit --omit=dev` → `found 0 vulnerabilities`.

If `npm audit --omit=dev` still reports advisories that only clear with
`npm audit fix --force`, STOP and report (forced major bumps are out of scope —
see STOP conditions).

### Step 3: Confirm the build still compiles

**Verify**: `npx tsc --noEmit` → exit 0, no output.

### Step 4: Confirm no test regression

**Verify**: `npx jest 2>&1 | tail -5` → the **same** counts you recorded in
Step 1 (`2 failed, 22 skipped, 532 passed`). The two failures are pre-existing
(see plan 002) — they must not increase, and passing/skipped counts must not
drop.

## Test plan

No new tests. This plan's correctness is proven by: (a) `npm audit --omit=dev`
going to zero, and (b) the existing suite showing identical counts before and
after. If `simple-git` or `ws` had a breaking API change, existing tests in
`__tests__/visual/baseline.test.ts` and `__tests__/protocol.test.ts` would
catch it.

## STOP conditions

Stop and report back (do not improvise) if:

- `npm audit --omit=dev` cannot reach zero without `npm audit fix --force`
  (a forced major-version bump is a separate decision the operator must make).
- The test suite shows MORE than 2 failures, or fewer than 532 passing, after
  the upgrade (the dependency bump broke something — report which tests).
- `npx tsc --noEmit` reports errors after the upgrade (a type-level breaking
  change in `simple-git`/`ws` typings).
